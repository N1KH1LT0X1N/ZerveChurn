# 01 - Data Loading and Overview

## Purpose
This notebook provides an initial overview of the user retention dataset, including:
- Dataset dimensions and structure
- Date ranges and temporal coverage
- Column types and data quality checks
- Memory usage analysis
- Missing data assessment

## Dataset Context
The `user_retention.parquet` file contains behavioral event data tracking user interactions with the Zerve platform. Each row represents a user event with associated metadata including timestamps, event types, user identifiers, and contextual information.

## Key Objectives
1. Load the parquet dataset efficiently
2. Understand the basic structure and size
3. Identify data types and schema
4. Check for data quality issues
5. Document temporal coverage
6. Assess memory footprint
